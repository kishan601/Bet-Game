1. Project Overview
You are required to develop a betting-based card matching web
application. The platform will allow users to place bets, play a
number-matching game, and win rewards based on outcomes.
The application will include frontend (React.js), backend (Node.js),
and database (MongoDB) integration along with proper game logic
and user management.

2. Game Functionality
Game Structure
● Total 10 cards displayed on screen
● Arranged in 2 rows (5 cards each)
Card Logic
● Each card contains a random number
● Example:
○ Row 1: 23, 45, 67, 98, 90
○ Row 2: 67, 98, 23, 90, 45

Gameplay Rules
● When user clicks on cards:
○ Numbers will shuffle/switch randomly
○ This prevents cheating
● If numbers match between rows:
○ Cards will remain open

● If not matched:
○ Cards will close again
● A timer-based switching system should be implemented to
ensure fairness

3. Betting System

Game Logic (Important Section – Must Follow)
Number Distribution
● Total 10 cards (2 rows × 5 cards)
● Each row will contain unique numbers (no duplicates within
same row)
● Both rows will contain same set of numbers, but in different
order
Example:
● Row 1: 23, 45, 67, 98, 90
● Row 2: 67, 98, 23, 90, 45

Card Click Behavior
● When user clicks on a card:
○ The number is revealed
● User selects one card from Row 1 and one from Row 2

Matching Logic
● If both selected cards have same number:
○ Cards will remain open (matched)
● If numbers do not match:
○ Cards will close again

Dynamic Number Switching (Anti-Cheat Logic) ⚠️
To prevent cheating, implement the following:
● After every card interaction:
○ Numbers in cards should shuffle/switch positions
dynamically

● Example:
○ User clicks and identifies 98
○ On next click → 98 should move to a different position
● This ensures:
○ User cannot memorize positions

Important Rules
● Numbers must NOT repeat within the same row
● Numbers must be same in both rows (for matching purpose)
● Positions must change dynamically after interactions

Session Behavior
● If user continues playing:
○ Numbers keep switching dynamically
● If user refreshes:
○ Generate completely new random numbers

Timer Logic
● Implement auto-switch/shuffle mechanism
● Even if user is inactive:
○ Numbers should keep changing at intervals
● This ensures fair gameplay
